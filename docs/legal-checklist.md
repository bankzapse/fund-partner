# รายการตรวจสอบด้านกฎหมายและความปลอดภัย

> **คำเตือนสำคัญ**
> เอกสารนี้จัดทำโดยผู้พัฒนาซอฟต์แวร์ **ไม่ใช่คำแนะนำทางกฎหมาย บัญชี หรือภาษี**
> จัดทำขึ้นเพื่อให้ทนายความและผู้สอบบัญชีของคุณใช้ตรวจสอบต่อได้เร็วขึ้น
> ตรงตามที่ SRS ข้อ 20 กำหนดไว้ว่าต้องให้ผู้เชี่ยวชาญตรวจก่อนเปิดใช้งานจริง
>
> ปรับปรุงล่าสุด: 20 กรกฎาคม 2569

---

## 1. สิ่งที่ระบบ "ทำแล้ว" ในเชิงเทคนิค

รายการเหล่านี้ทดสอบแล้วว่าทำงานจริง (ชุดทดสอบ 93 ข้อ ผ่านทั้งหมด)

| เรื่อง | สถานะ | หลักฐาน |
|---|---|---|
| รหัสผ่านเข้ารหัสด้วย bcrypt ไม่เก็บแบบอ่านได้ | ✅ | `lib/auth.js` · ทดสอบว่า API ไม่ส่ง hash ออกเลย |
| ออกจากระบบอัตโนมัติเมื่อไม่ใช้งาน | ✅ | ตั้งค่าได้ที่หน้าตั้งค่า (ค่าเริ่มต้น 120 นาที) |
| เชื่อมต่อผ่าน HTTPS พร้อม HSTS | ✅ | ตรวจจากภายนอกแล้ว |
| ไม่ลบรายการการเงินถาวร ใช้การยกเลิก | ✅ | ทดสอบว่าเลขที่ใบรับเงินไม่มีเลขขาดหาย |
| บันทึกผู้ทำรายการ เวลา ค่าเดิม ค่าใหม่ เหตุผล | ✅ | `audit_logs` · ทดสอบครบทุกการเคลื่อนไหว |
| แยกสิทธิ์ 4 ตำแหน่งตาม SRS ข้อ 12 | ✅ | ทดสอบการยกระดับสิทธิ์ 8 แบบ ถูกปฏิเสธทั้งหมด |
| พนักงานเก็บเงินเห็นเฉพาะลูกหนี้ของตน | ✅ | ทดสอบการเดา id ข้ามเขต ถูกปฏิเสธ |
| ป้องกัน SQL Injection | ✅ | ยิง payload 5 แบบ ไม่มีข้อมูลรั่วหรือเสียหาย |
| ไฟล์แนบจำกัดชนิดและสุ่มชื่อไฟล์ใหม่ | ✅ | ปฏิเสธ .exe .html .svg .js · กัน path traversal |
| ไฟล์สำรองข้อมูลไม่มีรหัสผ่านติดไป | ✅ | ทดสอบแล้ว |
| ฐานข้อมูลเปิด Row Level Security ครบทุกตาราง | ✅ | 16/16 ตาราง |
| ปิด Data API สาธารณะของ Supabase | ✅ | ตารางการเงินไม่เปิดให้เข้าถึงจากเบราว์เซอร์โดยตรง |
| ไม่ส่งข้อมูลผู้เข้าชมไปบุคคลที่สาม | ✅ | รูปภาพเก็บบนเซิร์ฟเวอร์เราเอง ไม่มี tracker |

## 2. สิ่งที่ต้องให้ผู้เชี่ยวชาญตรวจก่อนใช้กับลูกค้าจริง

### 2.1 อัตราดอกเบี้ยและค่าธรรมเนียม 🔴 สำคัญที่สุด

ระบบ **ไม่ได้บังคับเพดานดอกเบี้ยใด ๆ** — ผู้ใช้กรอกตัวเลขได้อิสระตามที่ SRS ข้อ 7.1 กำหนด
และ SRS ข้อ 20 ระบุชัดว่าห้ามฝังอัตราแบบแก้ไขไม่ได้

**สิ่งที่ต้องตรวจกับทนาย**

- ธุรกิจของคุณเข้าข่ายกฎหมายฉบับใด — พิจารณาอย่างน้อย
  - ประมวลกฎหมายแพ่งและพาณิชย์ (การกู้ยืมเงินทั่วไป)
  - พ.ร.บ. ห้ามเรียกดอกเบี้ยเกินอัตรา พ.ศ. 2560
  - ประกาศธนาคารแห่งประเทศไทย หากเข้าข่ายสินเชื่อที่อยู่ภายใต้การกำกับ
    (เช่น สินเชื่อส่วนบุคคล / นาโนไฟแนนซ์ / พิโกไฟแนนซ์ ซึ่งต้องขอใบอนุญาต)
- อัตราที่คุณจะตั้งจริง เมื่อคิดเป็น **อัตราต่อปี (APR)** แล้วเกินเพดานที่กฎหมายกำหนดหรือไม่
- **ค่าทำเอกสาร 100 บาท** ที่ระบบหักอัตโนมัติ — ตามกฎหมายไทย ค่าธรรมเนียมบางประเภท
  ถูกนับรวมเป็นดอกเบี้ยในการคำนวณเพดาน ต้องให้ทนายยืนยันว่าคิดได้และคิดอย่างไร

> ⚠️ ตัวอย่างในเอกสาร SRS (ค่างวด 50 บาท แบ่งเป็นดอก 20 เงินต้น 30 ต่อวัน)
> เมื่อคิดเป็นอัตราต่อปีจะสูงมาก **ต้องให้ทนายตรวจก่อนใช้จริงอย่างยิ่ง**
> ระบบคำนวณตามที่คุณกรอก ไม่ได้รับรองว่าตัวเลขนั้นถูกกฎหมาย

### 2.2 ใบอนุญาตประกอบธุรกิจ

- ปล่อยกู้เป็นธุรกิจโดยไม่มีใบอนุญาตอาจผิดกฎหมาย ขึ้นกับลักษณะและขนาดของกิจการ
- ตรวจว่าต้องขึ้นทะเบียนกับกระทรวงการคลังหรือ ธปท. หรือไม่

### 2.3 PDPA — พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล 🟠

ระบบเก็บข้อมูลส่วนบุคคลของลูกหนี้: ชื่อ เบอร์โทร ที่อยู่ **สำเนาบัตรประชาชน** และรูปถ่าย

สิ่งที่ระบบเตรียมไว้ให้แล้ว

- ควบคุมการเข้าถึงตามบทบาท พนักงานเห็นเฉพาะลูกหนี้ของตน
- บันทึกร่องรอยว่าใครเข้าถึงและแก้ไขอะไร
- ไฟล์แนบเก็บแยกและไม่เปิดให้เดา URL ได้ง่าย

สิ่งที่ **ยังต้องทำเพิ่มเอง** (ระบบทำแทนไม่ได้)

| ต้องทำ | หมายเหตุ |
|---|---|
| หนังสือให้ความยินยอม (Consent) จากลูกหนี้ | ต้องระบุวัตถุประสงค์การเก็บและระยะเวลาเก็บ |
| ประกาศความเป็นส่วนตัว (Privacy Notice) | ควรมีหน้าสาธารณะให้ลูกค้าอ่านได้ |
| กำหนดระยะเวลาเก็บข้อมูลและวิธีทำลาย | ระบบยังไม่มีการลบข้อมูลอัตโนมัติเมื่อครบกำหนด |
| ผู้ควบคุมข้อมูลส่วนบุคคล และช่องทางให้ลูกค้าใช้สิทธิ์ | สิทธิ์ขอเข้าถึง แก้ไข ลบ ตามกฎหมาย |
| ข้อตกลงกับผู้ประมวลผลข้อมูล (DPA) | Supabase และ Vercel เป็นผู้ประมวลผลแทนคุณ |
| แจ้งว่าข้อมูลถูกเก็บที่สิงคโปร์ | การโอนข้อมูลออกนอกประเทศมีเงื่อนไขตาม PDPA |

> **ข้อควรพิจารณา:** ปัจจุบันข้อมูลเก็บที่ Supabase region สิงคโปร์
> ถ้าที่ปรึกษากฎหมายเห็นว่าต้องเก็บในประเทศไทย ต้องย้ายผู้ให้บริการ
> ระบบย้ายได้เพราะใช้ PostgreSQL มาตรฐาน ไม่ผูกกับผู้ให้บริการรายใด

### 2.4 การทวงถามหนี้ 🟠

**พ.ร.บ. การทวงถามหนี้ พ.ศ. 2558** กำหนดข้อห้ามไว้ชัดเจน เช่น

- ห้ามทวงถามกับบุคคลอื่นที่ไม่ใช่ลูกหนี้
- จำกัดเวลาและความถี่ในการติดต่อ
- ห้ามใช้ความรุนแรง ข่มขู่ หรือทำให้อับอาย

ระบบมีรายงานลูกหนี้ค้างชำระและข้อมูลติดต่อ — **วิธีที่พนักงานนำไปใช้ต้องอยู่ในกรอบกฎหมาย**
ควรกำหนดระเบียบปฏิบัติภายในและอบรมพนักงานเก็บเงิน

### 2.5 ภาษีและบัญชี 🟡

- รายได้ดอกเบี้ยและค่าธรรมเนียมต้องนำส่งภาษีอย่างไร
- ต้องออกใบกำกับภาษีหรือไม่ (ระบบออกได้แค่ใบรับเงิน)
- ต้องหักภาษี ณ ที่จ่ายในกรณีใดบ้าง
- รูปแบบเอกสารที่กรมสรรพากรยอมรับ

> ระบบส่งออกข้อมูลเป็น Excel/CSV และ JSON ได้ครบ ผู้สอบบัญชีนำไปใช้ต่อได้

### 2.6 รูปแบบสัญญา 🟡

ระบบสร้าง **ตารางการชำระ** ให้ แต่ **ไม่ได้สร้างตัวสัญญาที่มีผลทางกฎหมาย**
ต้องให้ทนายร่างแบบฟอร์มสัญญาแยกต่างหาก และตรวจว่าข้อความในใบรับเงินที่ระบบพิมพ์
เพียงพอเป็นหลักฐานการชำระหรือไม่

## 3. ความปลอดภัย — สิ่งที่ควรทำเพิ่ม

เรียงตามความสำคัญ ระบบใช้งานได้แล้วโดยไม่มีข้อเหล่านี้ แต่ควรทยอยทำ

| ลำดับ | เรื่อง | เหตุผล |
|---|---|---|
| 1 | **จำกัดจำนวนครั้งการล็อกอินผิด** | ตอนนี้ยังเดารหัสผ่านซ้ำได้ไม่จำกัด ควรล็อกชั่วคราวหลังผิดหลายครั้ง |
| 2 | **ยืนยันตัวตนสองชั้น (2FA)** สำหรับบัญชีเจ้าของ | บัญชีเจ้าของเข้าถึงข้อมูลการเงินทั้งหมด |
| 3 | **ทดสอบกู้คืนข้อมูลจริง** | Supabase สำรองอัตโนมัติ แต่ควรลองกู้จริงอย่างน้อยปีละครั้ง |
| 4 | จำกัดจำนวนคำขอต่อนาที (rate limit) | กันการยิงถล่มและการดูดข้อมูล |
| 5 | แจ้งเตือนเมื่อมีการล็อกอินจากอุปกรณ์ใหม่ | ตรวจจับบัญชีถูกขโมยได้เร็วขึ้น |
| 6 | นโยบายความซับซ้อนของรหัสผ่าน | ตอนนี้บังคับแค่ 6 ตัวอักษรขึ้นไป |

### สิ่งที่ต้องทำทันทีเมื่อมีพนักงานใช้งาน

- สร้างบัญชีแยกให้ทุกคน **ห้ามใช้บัญชีร่วมกัน** เพราะ Audit Log จะไร้ความหมาย
- ผูกพนักงานเก็บเงินกับข้อมูลพนักงานในระบบ เพื่อให้ขอบเขตการมองเห็นทำงาน
- ปิดบัญชีทันทีเมื่อพนักงานลาออก (ระบบตัดสิทธิ์ทันทีที่ปิด ทดสอบแล้ว)

## 4. ข้อจำกัดของระบบที่ควรรู้

| เรื่อง | รายละเอียด |
|---|---|
| ไม่ตรวจเพดานดอกเบี้ย | ระบบคำนวณตามที่กรอก ไม่เตือนว่าเกินกฎหมายหรือไม่ |
| ไม่สร้างตัวสัญญา | สร้างแค่ตารางงวดและใบรับเงิน |
| ไม่มีการลบข้อมูลตามกำหนดเวลา | ต้องจัดการเองตามนโยบาย PDPA ที่กำหนด |
| ไม่ใช่ระบบบัญชีเต็มรูปแบบ | ไม่มีงบดุล ไม่มีผังบัญชี ใช้คู่กับโปรแกรมบัญชีหรือผู้สอบบัญชี |
| ข้อมูลอยู่ที่สิงคโปร์ | ย้ายได้ถ้าจำเป็น เพราะใช้ PostgreSQL มาตรฐาน |

## 5. คำถามที่ควรถามทนายความ

คัดลอกส่งให้ทนายได้เลย

1. ธุรกิจของเราเข้าข่ายต้องขอใบอนุญาตปล่อยสินเชื่อหรือไม่ ถ้าใช่ ประเภทใด
2. อัตราดอกเบี้ยที่เราตั้งไว้ เมื่อคิดเป็นอัตราต่อปีแล้วเกินเพดานตามกฎหมายหรือไม่
3. ค่าทำเอกสาร 100 บาทต่อสัญญา ต้องนับรวมเป็นดอกเบี้ยในการคำนวณเพดานหรือไม่
4. การหักงวดแรกออกจากเงินที่จ่ายให้ลูกค้าทันที มีประเด็นทางกฎหมายหรือไม่
5. การ "รียอด" (ปิดสัญญาเดิมแล้วตั้งสัญญาใหม่จากยอดคงเหลือ) ทำได้หรือไม่ ต้องระบุอะไรในสัญญา
6. เราต้องทำเอกสาร PDPA อะไรบ้าง และเก็บสำเนาบัตรประชาชนได้ภายใต้เงื่อนไขใด
7. การเก็บข้อมูลลูกค้าไว้ที่เซิร์ฟเวอร์ในสิงคโปร์ ทำได้หรือไม่ ต้องแจ้งลูกค้าอย่างไร
8. ระเบียบการทวงถามหนี้ที่พนักงานต้องปฏิบัติ ควรเขียนไว้อย่างไร
9. ใบรับเงินที่ระบบพิมพ์ ใช้เป็นหลักฐานทางกฎหมายได้หรือไม่ ต้องเพิ่มข้อความใด
10. ต้องเก็บเอกสารและข้อมูลย้อนหลังกี่ปี
